# 本地多分支开发测试指南

> 适用场景：本机同时开发/测试多个 Issue，或对多个 PR 进行联合合测（integration test），不影响主目录运行的前提下独立跑前端/小程序 dev server。

---

> **【强制约束】feature / fix 分支必须从 `develop` 拉取；`patch` / `hotfix` 从 `main` 创建（见 AGENTS.md 分支模型）。**
> `main` 是生产分支，只接受 `patch` PR、`hotfix` PR 和 dev 验收通过的 release PR。

---

## 1. 核心概念

| 概念 | 说明 |
|---|---|
| **主目录** | `~/Code/PSA`，通常跑在端口 18000（PC）或由 `local-dev-tools/` 脚本管理 |
| **Worktree** | 同一个 Git 仓库在磁盘上的另一个工作目录，有自己的分支，互不干扰 |
| **`.worktrees/`** | 约定的 worktree 存放路径，已加入 `.gitignore`，**不会上传到 GitHub** |
| **端口规则** | 主目录 18000；`issue-N` 分支 worktree 自动映射到 `8000 + (N % 1000)`（如 issue-220 → 8220） |
| **`develop` 是唯一基准** | **所有 feature / fix / test 分支必须从 `develop` 拉取，`main` 是生产分支，任何人不得以 `main` 为基础创建本地分支** |

---

## 2. 前提条件

- macOS，已安装 git / pnpm / Node.js（nvm 管理）
- 本地后端服务已启动（通过 `local-dev-tools/start-backend-local.command`）
- 主目录已有 `psa-front-desk-pc/.env.local`（本地联调配置文件，不提交 git）
- 微信开发者工具已安装（小程序测试才需要）

---

## 3. Worktree 目录结构

```
~/Code/PSA/                         ← 主目录（ develop / fix 分支）
├── .worktrees/                     ← 所有 worktree 放这里（已 gitignore）
│   ├── issue-220/                  ← 单 issue worktree
│   └── integration-220-221/        ← 多 issue 合测 worktree
└── scripts/
    ├── serve-worktree.sh           ← PC 端 worktree 启动脚本
    └── serve-mini-worktree.sh      ← 小程序 worktree 启动脚本
```

> `.worktrees/` 已在根 `.gitignore` 中，只存在于本地磁盘，**不会被 git 跟踪，也不会上传 GitHub**。

---

## 4. 单分支测试

> **【强制】分支基准必须是 `develop`，禁止从 `main` 创建。**

### 4.1 创建 Worktree

```bash
# 在主目录根下执行（PSA/）
# 语法：git worktree add <路径> -b <新分支名> <基准分支>

# 新建分支并创建 worktree（必须显式指定 develop 作为基准）
git worktree add .worktrees/issue-220 -b fix/issue-220-xxx develop

# 已有远程分支时，直接 checkout 到 worktree（分支已存在，无需指定基准）
git worktree add .worktrees/issue-220 fix/issue-220-xxx
```

### 4.2 准备 .env.local（每个 worktree 只需做一次）

```bash
# 从主目录复制本地联调配置
cp psa-front-desk-pc/.env.local \
   .worktrees/issue-220/psa-front-desk-pc/.env.local
```

> `.env.local` 包含 API 地址、开关项等，已被 `.gitignore` 忽略，每人在自己机器维护。

### 4.3 安装前端依赖（worktree 首次使用时）

```bash
cd .worktrees/issue-220/psa-front-desk-pc
pnpm install
cd ..   # 回到 worktree 根目录
```

> pnpm 使用全局 store，与主目录共享，安装速度很快。

### 4.4 启动 PC 端

```bash
# 在 worktree 根目录（.worktrees/issue-220/）
./scripts/serve-worktree.sh

# 输出示例：
#   Branch : fix/issue-220-pc-performance
#   Port   : http://localhost:8220
```

端口由脚本自动计算（`issue-220` → `8220`），通过环境变量注入，不修改任何配置文件。  
手动指定端口：`./scripts/serve-worktree.sh 8123`

### 4.5 启动小程序（如需）

```bash
# 在 worktree 根目录（.worktrees/issue-220/）
./scripts/serve-mini-worktree.sh

# 或手动指定后端 LAN IP：
./scripts/serve-mini-worktree.sh 192.168.6.100
```

产物输出到：`.worktrees/issue-220/psa-mini-program-new/dist/dev/mp-weixin`  
在微信开发者工具中「导入项目」，选择上述目录。

## 5. 多分支合测（Integration Test）

当需要同时验证多个 Issue 联合工作时，创建一个 integration 分支，合入所有相关分支后一起测试。

> **【强制】integration 分支必须从 `develop` 拉取，禁止从 `main` 创建。**

### 5.1 创建 Integration 分支

```bash
# 命名约定：test/integration-{issue1}-{issue2}-...
# 必须基于 develop

git checkout develop
git pull origin develop          # 先拉取最新 develop，避免基于旧版本
git checkout -b test/integration-220-221
git merge fix/issue-220-xxx
git merge fix/issue-221-xxx
# 如有冲突，手动解决后 git add / git commit
```

### 5.2 创建合测 Worktree

```bash
# 在主目录根下
git worktree add .worktrees/integration-220-221 test/integration-220-221
```

### 5.3 准备 .env.local 并安装依赖

```bash
cp psa-front-desk-pc/.env.local \
   .worktrees/integration-220-221/psa-front-desk-pc/.env.local

cd .worktrees/integration-220-221/psa-front-desk-pc
pnpm install
cd ../..
```

### 5.4 启动合测 PC 端

Integration 分支名含多个 issue 编号，脚本取第一个计算端口（`test/integration-220-221` → `8220`）。  
若与其他 worktree 端口冲突，手动指定：

```bash
./scripts/serve-worktree.sh 8900
```

### 5.5 启动合测小程序（如需）

```bash
./scripts/serve-mini-worktree.sh
```

产物目录：`.worktrees/integration-220-221/psa-mini-program-new/dist/dev/mp-weixin`  
在微信开发者工具中切换到此项目路径。

### 5.6 合测完成后清理

```bash
# 删除合测 worktree（回到主目录）
git worktree remove .worktrees/integration-220-221

# 删除合测分支（可选）
git branch -D test/integration-220-221
```

## 6. 端口速查表

| 场景 | 端口 | 启动方式 |
|---|---|---|
| 主目录日常开发 | 18000 | `local-dev-tools/start-pc-local.command` |
| Worktree issue-N | 8000 + (N % 1000) | `./scripts/serve-worktree.sh` |
| Worktree issue-220 | 8220 | 自动推断 |
| Worktree issue-55 | 8055 | 自动推断 |
| Integration 合测 | 建议 8900 | `./scripts/serve-worktree.sh 8900` |
| 手动指定端口 | 任意 | `./scripts/serve-worktree.sh <端口>` |

> 小程序无端口概念，多个 worktree 的产物在各自 `dist/dev/mp-weixin` 目录，微信开发者工具切换项目路径即可区分。

## 7. 常见问题

**Q：脚本报 `.env.local` 不存在怎么办？**  
A：每个新 worktree 首次使用前需手动从主目录复制：  
`cp <主目录>/psa-front-desk-pc/.env.local ./.worktrees/<issue>/psa-front-desk-pc/.env.local`

**Q：端口冲突了怎么办？**  
A：`./scripts/serve-worktree.sh <其他端口>` 手动指定即可，端口写入临时环境变量，不影响文件。

**Q：Worktree 会不会污染主目录的 git 历史？**  
A：不会。Worktree 是独立的工作目录和分支，`.worktrees/` 已在 `.gitignore` 中，git status / add / commit 全程看不到它。

**Q：多人都创建了 issue-220 的 worktree，会冲突吗？**  
A：不会。每个人的 `.worktrees/` 都在自己本机，不上 GitHub，互不影响。

**Q：已有的 sibling worktree（如 `PSA-issue-110/`）怎么处理？**  
A：已有的继续可用，`serve-worktree.sh` 脚本同样适用于 sibling worktree，只需在该目录下执行即可。新的 worktree 建议统一放在 `.worktrees/` 下便于管理。

**Q：小程序如何区分不同 worktree 的产物？**  
A：微信开发者工具「导入项目」时选择对应 worktree 的 `psa-mini-program-new/dist/dev/mp-weixin` 路径。不同 worktree 是不同目录，不会互相覆盖。

## 8. 快速参考：完整操作序列

### 单 Issue 完整流程

```bash
# ① 拉取最新 develop（基准分支）
git fetch origin
git checkout develop && git pull origin develop

# ② 创建 worktree（必须显式指定 develop 作为基准）
git worktree add .worktrees/issue-220 -b fix/issue-220-xxx develop

# ③ 复制 .env.local
cp psa-front-desk-pc/.env.local .worktrees/issue-220/psa-front-desk-pc/.env.local

# ④ 安装依赖（首次）
cd .worktrees/issue-220/psa-front-desk-pc && pnpm install && cd ../..

# ⑤ 启动 PC（端口 8220）
.worktrees/issue-220/scripts/serve-worktree.sh

# ⑥ 启动小程序（可选）
.worktrees/issue-220/scripts/serve-mini-worktree.sh
```

### 多 Issue 合测完整流程

```bash
# ① 拉取最新 develop
git fetch origin
git checkout develop && git pull origin develop

# ② 创建合测分支并合入各 issue（必须基于 develop）
git checkout -b test/integration-220-221
git merge fix/issue-220-xxx
git merge fix/issue-221-xxx

# ② 创建 worktree
git worktree add .worktrees/integration-220-221 test/integration-220-221

# ③ 复制 .env.local + 安装依赖
cp psa-front-desk-pc/.env.local .worktrees/integration-220-221/psa-front-desk-pc/.env.local
cd .worktrees/integration-220-221/psa-front-desk-pc && pnpm install && cd ../../..

# ④ 启动（端口 8900 避免与单 issue worktree 冲突）
.worktrees/integration-220-221/scripts/serve-worktree.sh 8900

# ⑤ 测试完成，清理
git worktree remove .worktrees/integration-220-221
git branch -D test/integration-220-221
```
